# 每日整理任务检查清单

## 执行前检查
- [ ] 检查是否有超过10MB的会话文件需要特殊处理
- [ ] 确认时间窗口正确（昨天5:00到今天5:00）

## 执行后检查
- [ ] 查看处理日志，确认处理的文件数量和消息数量
- [ ] 对比飞书消息数量和处理保存数量，差异超过10条需排查
- [ ] 检查GitHub推送状态，确保提交成功推送到远程
- [ ] 查看 vault 中新创建的对话文件数量

## 每周检查
- [ ] 检查 `.learnings/ERRORS.md` 是否有新错误
- [ ] 检查是否有未推送的Git提交
- [ ] 验证会话文件清理是否正常（保留7天）

## 问题响应
- 用户反馈记录缺失时，立即：
  1. 检查对应日期的会话文件是否存在
  2. 检查文件大小是否超过限制
  3. 手动提取并补处理
  4. 记录到 `.learnings/MANUAL_FIXES.md`
