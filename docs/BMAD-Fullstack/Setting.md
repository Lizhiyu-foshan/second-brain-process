# BMAD-Fullstack 项目初始化指南

## 概述

本文档指导你从零开始初始化一个使用 BMAD-Fullstack 的全栈项目，包含目录结构、配置和基础代码。

---

## 第一步：创建项目目录结构

```bash
# 创建项目根目录
mkdir my-fullstack-app
cd my-fullstack-app

# 初始化 Git
git init

# 创建目录结构
mkdir -p backend/app/{schemas,models,routers,services}
mkdir -p backend/tests
mkdir -p frontend/src/{types,api,components,pages}
mkdir -p frontend/public
mkdir -p .bmad/{templates,logs,features}

# 创建基础文件
touch README.md
touch .gitignore
```

### 最终目录结构

```
my-fullstack-app/
├── backend/                 # FastAPI 后端
│   ├── app/
│   │   ├── __init__.py
│   │   ├── main.py         # FastAPI 入口
│   │   ├── schemas/        # Pydantic Schema
│   │   ├── models/         # SQLAlchemy 模型
│   │   ├── routers/        # API 路由
│   │   └── services/       # 业务逻辑
│   ├── tests/
│   └── requirements.txt
├── frontend/               # React 前端
│   ├── src/
│   │   ├── types/          # 生成的 TS 类型
│   │   ├── api/            # API 请求
│   │   ├── components/     # 组件
│   │   └── pages/          # 页面
│   ├── package.json
│   └── vite.config.ts
├── .bmad/                  # BMAD-Fullstack 配置
│   ├── project.yaml        # 项目配置
│   ├── api-contract.yaml   # API 契约文档
│   ├── features/           # 功能需求文档
│   └── templates/          # 约束模板
└── README.md
```

## 第二步：配置 BMAD-Fullstack

创建 `.bmad/project.yaml`：

```yaml
project:
  name: "My Fullstack App"
  description: "用户管理系统"
  version: "1.0.0"

backend:
  framework: "fastapi"
  schema_path: "backend/app/schemas"
  main_file: "backend/app/main.py"
  port: 8000

frontend:
  framework: "react"
  type_path: "frontend/src/types"
  api_client_path: "frontend/src/api"
  port: 5173

contract:
  naming_convention: "camelCase"  # 前端使用 camelCase
  check_field_names: true
  check_field_types: true
  check_required: true
  strict_mode: true

# 生成配置
generation:
  auto_generate_types: true
  output_file: "frontend/src/types/api.ts"
  backup_existing: true
```

## 第三步：初始化后端 (FastAPI)

```bash
cd backend

# 创建 requirements.txt
cat > requirements.txt << 'EOF'
fastapi==0.109.0
uvicorn[standard]==0.27.0
pydantic==2.5.0
pydantic[email]==2.5.0
sqlalchemy==2.0.25
python-multipart==0.0.6
pytest==7.4.0
httpx==0.26.0
EOF

# 安装依赖
pip install -r requirements.txt

cd ..
```

### 创建后端入口文件

`backend/app/main.py`：

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

app = FastAPI(
    title="My Fullstack App",
    description="用户管理系统 API",
    version="1.0.0"
)

# CORS 配置
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5173"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

@app.get("/")
async def root():
    return {"message": "API is running"}
```

## 第四步：创建基础 Schema

`backend/app/schemas/__init__.py`：

```python
from .user import UserCreate, UserResponse

__all__ = ["UserCreate", "UserResponse"]
```

`backend/app/schemas/user.py`：

```python
from pydantic import BaseModel, EmailStr, Field
from typing import Optional
from datetime import datetime

class UserCreate(BaseModel):
    """用户创建请求"""
    username: str = Field(..., min_length=3, max_length=50)
    email: EmailStr
    password: str = Field(..., min_length=6)
    age: Optional[int] = Field(None, ge=0, le=150)

class UserResponse(BaseModel):
    """用户响应"""
    id: int
    username: str
    email: str
    is_active: bool = True
    created_at: datetime
```

## 第六步：生成前端类型

```bash

  python3 /path/to/bmad-fullstack/bmad-fullstack \
  generate-ts \
  --input backend/app/schemas \
  --output frontend/src/types/api.ts \
  --naming camelCase
```

## 第八步：创建启动脚本

`start-dev.sh`：

```bash
#!/bin/bash

GREEN='\033[0;32m'
BLUE='\033[0;34m'
NC='\033[0m'

echo -e "${BLUE}=== 启动开发环境 ===${NC}"

# 检查后端 Schema
echo -e "${BLUE}[1/4] 检查后端契约...${NC}"
PYTHONPATH=/path/to/bmad-fullstack/lib \
  python3 /path/to/bmad-fullstack/bmad-fullstack \
  check --backend backend/app/schemas

if [ $? -ne 0 ]; then
  echo "❌ 后端契约检查失败，请先修复问题"
  exit 1
fi

# 生成前端类型
echo -e "${BLUE}[2/4] 生成前端类型...${NC}"
PYTHONPATH=/path/to/bmad-fullstack/lib \
  python3 /path/to/bmad-fullstack/bmad-fullstack \
  generate-ts \
  --input backend/app/schemas \
  --output frontend/src/types/api.ts \
  --naming camelCase

# 启动后端
echo -e "${BLUE}[3/4] 启动后端服务...${NC}"
cd backend && uvicorn app.main:app --reload --port 8000 &
BACKEND_PID=$!
cd ..

# 启动前端
echo -e "${BLUE}[4/4] 启动前端服务...${NC}"
cd frontend && npm run dev &
FRONTEND_PID=$!
cd ..

echo -e "${GREEN}✅ 开发环境已启动！${NC}"
echo "后端: http://localhost:8000"
echo "前端: http://localhost:5173"
echo "API 文档: http://localhost:8000/docs"

trap "kill $BACKEND_PID $FRONTEND_PID; exit" INT
wait
```

```bash
chmod +x start-dev.sh
```

## 第九步：CI/CD 配置

`.github/workflows/contract-check.yml`：

```yaml
name: API Contract Check

on:
  push:
    branches: [main, develop]
    paths:
      - 'backend/app/schemas/**'
      - 'frontend/src/types/**'
  pull_request:
    branches: [main]

jobs:
  contract-check:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.11'
      
      - name: Install BMAD-Fullstack
        run: |
          cd /tmp
          git clone https://github.com/Lizhiyu-foshan/bmad-fullstack.git
          cd bmad-fullstack
          pip install -r requirements.txt
      
      - name: Check Backend Schemas
        run: |
          PYTHONPATH=/tmp/bmad-fullstack/lib \
            python3 /tmp/bmad-fullstack/bmad-fullstack check \
            --backend backend/app/schemas \
            --naming camelCase \
            --ci
      
      - name: Generate TypeScript Types
        run: |
          PYTHONPATH=/tmp/bmad-fullstack/lib \
            python3 /tmp/bmad-fullstack/bmad-fullstack generate-ts \
            --input backend/app/schemas \
            --output frontend/src/types/api.ts \
            --naming camelCase
      
      - name: Check Type Consistency
        run: |
          if [ -n "$(git status --porcelain frontend/src/types/api.ts)" ]; then
            echo "❌ 生成的类型文件有变更，请提交更新后的类型文件"
            exit 1
          fi
      
      - name: Set up Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '20'
      
      - name: Type Check Frontend
        run: |
          cd frontend
          npm ci
          npx tsc --noEmit
```
