# 定时任务系统修复计划 (2026-03-16)

## 背景

基于 7 天日志深度分析（2026-03-09 至 2026-03-16），发现定时任务系统存在**严重的系统性问题**。

---

## 关键发现

### 1. 重大事故
- **2026-03-13**: OpenClaw 版本更新导致所有定时任务失效
- **影响**: 6 个核心任务完全停止，用户严重不满
- **根本原因**: `isolated` + `agentTurn` 模式无法调用工具

### 2. 修复 - 复发循环
| 问题 | 修复次数 | 复发次数 |
|------|---------|---------|
| 飞书发送失败 | 3 | 2 |
| 路径错误 | 4 | 2 |
| AI 调用失败 | 3+ | 1+ |
| 重复消息 | 3 | 1 |

**平均**: 每个问题修复 3 次，复发 1.5 次

### 3. 修复引入新问题
- 修正路径 → 函数未定义
- 模型切换 → 上下文超限
- 迁移 Cron → 遗留 isolated 任务仍存风险

### 4. 失败频率
- 03-13 单日失败 **8 次**
- 7 天总计约 **20 次**
- 当前趋势：⚠️ **上升**（12 次/天）

---

## 根因分析

1. ❌ **缺乏预防性测试** - 修复后只做简单导入测试
2. ❌ **版本更新风险失控** - 无回滚机制
3. ❌ **配置分散难管理** - 路径分散在多个文件
4. ❌ **监控告警滞后** - 任务显示"成功"但实际未执行
5. ❌ **修复验证不充分** - 不等待下次任务执行确认

---

## 系统性修复方案

### 第一层：强制验证流程（立即执行）

**文档**: `/root/.openclaw/workspace/docs/fix_verification_checklist.md`

所有修复必须完成 6 步验证：
```
1. 修复代码
2. 单元测试
3. 集成测试
4. 完整流程验证（关键！）
5. 推送代码
6. 下次任务确认（最关键！）
```

**承诺规范**:
- ❌ 错误："明天应该能正常工作"
- ✅ 正确："已修复并验证，下次任务执行时间是 X 点，我会检查并向您汇报"

### 第二层：配置健康检查（今天完成）

**脚本**: `/root/.openclaw/workspace/scripts/config_health_check.py`

每次系统启动时自动验证：
- 所有路径配置是否有效
- 环境变量是否正确
- 关键文件是否存在
- Python 模块是否可导入
- Cron 任务状态

### 第三层：主动监控增强（本周完成）

**脚本**: `/root/.openclaw/workspace/scripts/task_execution_monitor.py`

不只是检查 cron status，还要：
- 检查文件时间戳（是否真正更新）
- 检查内容是否包含今天日期
- 检查执行时长是否异常
- 失败自动告警

### 第四层：架构优化（本月完成）

1. **统一配置管理**
   - 集中配置文件
   - 环境变量优先
   - 配置验证脚本

2. **遗留任务迁移**
   - 所有 isolated 模式任务迁移到 Linux Cron
   - 消除 OpenClaw 定时任务依赖

3. **技术债清理**
   - 重复代码合并
   - 废弃功能清理
   - 代码审查清单
