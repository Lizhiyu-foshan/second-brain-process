# 3月21日对话记录缺失问题分析报告

## 问题现象
- 2026-03-22 8:30 每日复盘报告显示"对话记录：❌ 未生成"
- 但 3月21日有大量会话文件（60+ 个）

## 根因分析

### 问题根源
`message_index.py` 的 `rebuild()` 方法存在**逻辑缺陷**：

1. **索引重建时**，它会扫描所有会话文件并找到最大时间戳 `max_timestamp`
2. **将 `last_processed` 设置为 `max_timestamp`**，表示"所有消息已处理"
3. **但实际对话文件并未生成**

### 触发条件
- 当索引文件损坏或校验和不匹配时，会触发自动重建
- 重建后的索引时间戳已经超前，导致增量处理器认为"没有新消息"

### 代码问题位置
```python
def rebuild(self) -> Dict:
    # ... 扫描所有消息 ...
    new_index = {
        "last_processed": {
            "timestamp_ms": max_timestamp,  # ← 问题：设置为最大时间戳
            # ...
        }
    }
```

### 长期修复建议

#### 方案1：分离"索引重建"和"处理状态"
```python
def rebuild(self) -> Dict:
    # 重建时只记录消息存在性，不假设已处理
    new_index = {
        "last_processed": {
            "timestamp_ms": 0,  # 重置为0，让增量处理器处理所有消息
        },
        "statistics": {
            "rebuilt_at": ...,  # 记录重建时间
        }
    }
```

#### 方案2：每日历史记录
在索引中添加 `daily_history`，记录每天是否已生成对话文件：
```json
{
  "daily_history": [
    {"date": "2026-03-21", "file": "2026-03-21_conversations.md", "message_count": 1824},
    {"date": "2026-03-20", "file": "2026-03-20_conversations.md", "message_count": ...}
  ]
}
```
