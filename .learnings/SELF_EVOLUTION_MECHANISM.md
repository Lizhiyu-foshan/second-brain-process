**核心思想**：从"被动响应"到"主动发现"，让规则系统自我进化


## 触发方式

### 1. 被动触发（异常发生时）

当检测到以下情况时立即触发：
- 同一类型错误7天内发生≥2次
- 修复操作未记录到规则溯源
- 规则健康度下降（<70分或误报≥3次）

### 2. 主动巡检（每日23:00）

定时任务自动运行：
```bash
python3 scripts/agents_self_evolution_trigger.py
```

### 3. 手动触发

用户可随时手动执行：
```bash
python3 scripts/agents_self_evolution_trigger.py
```

---

## 检测内容

| 检测类型 | 说明 | 严重程度 | 触发条件 |
|---------|------|---------|---------|
| **重复错误** | 同类型错误反复发生 | 🔴 高 | 7天内≥2次 |
| **未记录修复** | 修复未溯源到规则 | 🟡 中 | 7天内未记录 |
| **覆盖缺口** | 关键场景无规则 | 🟡 中 | 持续存在 |
| **健康度下降** | 规则效果变差 | 🔴 高 | 分数<70或误报≥3 |

## 执行流程

### Step 1: 异常检测
```python
trigger = SelfEvolutionTrigger()
anomalies = trigger.detect_anomalies()
```

### Step 2: 生成提案
```python
for anomaly in anomalies:
    proposal = trigger.generate_proposal(anomaly)
    trigger.save_proposal(anomaly)
```

### Step 3: 通知用户
```
🔔 AGENTS.md 自进化触发器发现异常

发现2个问题需要规则固化：
1. 🟡 3个场景无规则覆盖
2. 🔴 规则7健康度下降（72分，3次误报）

详细提案：.learnings/RULE_PROPOSALS.md

请确认是否执行根因分析和规则固化？
[确认] [查看详情] [忽略]
```

### Step 4: 用户确认
- **确认**：自动执行根因分析 → 规则设计 → 规则实现 → 验证测试
- **查看详情**：展示提案完整内容
- **忽略**：记录忽略原因，下次不再提醒

### Step 5: 规则固化
```
用户确认后自动执行：
1. 根因分析（收集日志、定位原因）
2. 规则设计（确定触发条件和流程）
3. 规则实现（更新AGENTS_DETAILS.md等）
4. 验证测试（模拟场景验证）
5. 更新溯源（记录事件和验证数据）
```

## 用户交互设计

### 主动通知时机

| 场景 | 通知方式 | 用户操作 |
|------|---------|---------|
| 发现重复错误 | 飞书消息 | 确认/查看/忽略 |
| 规则健康度下降 | 飞书消息 | 确认/查看/忽略 |
| 覆盖缺口 | 日报中提及 | 下次主动询问 |
| 每日巡检无异常 | 静默 | 无 |

### 确认后自动执行

用户回复"确认"后，Agent自动：
1. 读取提案文件
2. 执行根因分析
3. 生成规则草案
4. 向用户展示草案
5. 用户二次确认后实施

## 关键原则

1. **用户主导**：所有规则更新需用户确认，Agent不擅自修改
2. **透明可追溯**：所有提案、分析、决策都记录在案
3. **渐进进化**：小问题立即修复，大问题详细分析
4. **闭环验证**：每次更新后验证效果，反馈到健康度系统
