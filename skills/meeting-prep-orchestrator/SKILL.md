# Meeting Prep Orchestrator / Topic Discussion Igniter

讨论准备与知识预加载自动化 + 主题讨论激发器

## 核心升级

### 1. AI深度分析版（原有）
- 解析讨论主题
- 检索相关知识
- 生成准备摘要

### 2. 主题讨论激发器（新增） ⭐
- **主题发现**：自动从近期笔记中提取潜在讨论主题
- **AI深度分析**：读取内容、发现关联、生成洞察
- **渐进展现**：分4阶段抛出，激发思考
- **交互讨论**：根据用户回应深入探索
- **行动转化**：最终形成指引或工具

## 两种启动方式

### 方式A：主动发送"主题讨论"

**触发**：
```
主题讨论
```
或
```
今天有什么值得讨论的主题？
```

**系统行为**：
1. 扫描最近3天的笔记（memory/ + obsidian-vault/）
2. AI识别未完成的思考线索
3. 自动选择最值得讨论的主题
4. 分4阶段渐进展开：
   - 阶段1：引入主题，提出启动问题
   - 阶段2：深度展开核心洞察
   - 阶段3：发现与其他主题的关联
   - 阶段4：转化为行动指南或工具

### 方式B：发送相关链接时触发

**触发**：
用户发送任何链接

**系统行为**：
1. 分析链接内容与当前/近期主题的关联性
2. 如果相关，抛出关联指引：
   ```
   🔗 关联发现
   
   你分享的这个链接与「XX主题」相关。
   
   这让我想到：基于这个新材料，
   我们是否可以从YY角度重新审视？
   ```
3. 邀请用户一同讨论

## 解决的核心痛点

| 痛点 | 解决方案 |
|------|---------|
| 当天太忙，没展开讨论 | AI自动发现未完成的思考线索，主动提出讨论 |
| 发散思考缺乏关联性 | AI发现笔记间的隐藏关联，建立知识网络 |
| 没转化成指引/工具 | 阶段4专门设计行动转化，形成可落地成果 |
| 一次性信息过载 | 4阶段渐进展现，每步只聚焦一个问题 |

### 示例1：主动发现主题

**用户**：主题讨论

（继续回复，我们一起挖掘更多）

## 技术实现

### 文件位置
```
/root/.openclaw/workspace/skills/meeting-prep-orchestrator/
├── SKILL.md
├── scripts/
│   ├── prep_check.py              # 基础版：会议准备
│   └── topic_discussion_igniter.py # 升级版：主题激发 ⭐
```

### 核心组件

**主题发现** (`discover_topics_from_recent_notes`):
- 扫描最近3天笔记
- 识别发散思考标记（"发散"、"联想"、"疑问"、"TODO"等）
- 返回候选主题列表

**AI分析** (`analyze_topic_with_ai`):
- 读取相关笔记内容
- 提取核心洞察
- 设计讨论路径（4阶段）
- 生成开放式问题

**状态管理** (`TopicDiscussionIgniter`):
- 记录当前讨论主题
- 跟踪讨论阶段
- 保存对话历史
- 支持多轮交互

**渐进展现** (`generate_stage_message`):
- 阶段1：引入主题（激发兴趣）
- 阶段2：深度展开（核心洞察）
- 阶段3：关联发现（知识网络）
- 阶段4：行动转化（指引/工具）

## 配置

### 自动触发（可选）
添加到系统级cron，定期检查是否有值得讨论的主题：
```bash
# 每天下午检查一次
0 14 * * * cd ... && python3 .../topic_discussion_igniter.py --discover >> /tmp/topic_discussion.log 2>&1
```

### 手动触发
```bash
# 自动发现并启动
python3 .../topic_discussion_igniter.py --test

# 指定主题启动
python3 .../topic_discussion_igniter.py --topic "API设计"

# 处理用户回应
python3 .../topic_discussion_igniter.py --respond "我觉得应该..."
```
