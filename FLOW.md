# Second Brain 处理流程文档

## 核心文件

| 文件 | 功能 |
|------|------|
| `processor.py` | 基础处理函数（队列管理、文件解析） |
| `ai_processor.py` | AI 内容提炼 |
| `git_sync.py` | Git 双向同步（含3次重试） |
| `process_all.py` | **统一处理入口** |
| `daily_task.py` | 定时任务（待确认列表、聊天记录归纳） |
| `daily_report.py` | 每日复盘报告 |
| `link_handler.py` | 链接处理（选项提示） |

## 处理流程

### 1. 用户发送链接

```
用户发送链接
    ↓
显示选项：
  【1】存入待读列表 ⭐默认
  【2】直接聊聊这个主题
  【3】搜索其他相关讨论
  ⏰ 15分钟无回复，默认【1】
    ↓
如果选择【1】：
  - 保存到 queue/
  - 检查队列数量
  - 提示是否需要立即处理
```

### 2. 批量处理（A1/A2/A3）

```
用户选择 A1/A2/A3
    ↓
process_all.py --batch --mode [full|summary|brief]
    ↓
逐篇处理：
  📄 第 1/N 篇处理完成：
  ✅ 《文章标题》
  📊 文章长度：XXX 字
  📝 开头预览：...（或 Key Takeaway + 核心要点）
  ---
    ↓
双向同步：
  🔄 开始双向同步到 GitHub...
  - git add -A
  - git commit -m "message"
  - git fetch origin main
  - git merge origin/main
  - git push origin main（失败重试3次）
    ↓
反馈结果：
  ✅ 已成功同步到 GitHub
  或
  ❌ 同步失败（已重试3次）
  💡 建议操作：
     1. 检查网络连接
     2. 手动运行：cd ... && git push
     3. 或稍后再试
```

### 3. 差异化处理（B + 1/2/3）

```
用户选择 B
    ↓
逐篇展示：
  📄 第 1/N 篇
  标题：xxx
  作者：xxx
  预览：...
  请选择：1.原文保存 / 2.主体+核心 / 3.精简摘要
    ↓
用户回复 1/2/3
    ↓
process_all.py --mode [full|summary|brief]
    ↓
（同批量处理的逐篇展示+同步流程）
```

### 4. 定时任务（每晚 23:30）

```
检查两类任务：
  1. 聊天记录主题归纳（过去24小时）
     - 自动分类
     - 提示是否保留（Y=保留，N=删除）⭐默认Y
     - ⏰ 5分钟超时，默认Y
  2. 待读笔记列表
     - 显示待处理笔记
     - 选择处理方式
     - ⏰ 15分钟超时，默认A2
    ↓
如果聊天记录选择Y：
  - 转换为笔记加入队列
  - 继续处理待读笔记（包含新增的）
    ↓
执行处理流程（同上述批量/差异化处理）
```

## 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| 聊天记录超时 | 5分钟 | 快速决策，不阻塞后续 |
| 待读笔记超时 | 15分钟 | 充足时间考虑处理方式 |
| Git 同步重试 | 3次 | 每次间隔5秒 |
| 聊天记录默认 | Y（保留） | 避免误删 |
| 待读笔记默认 | A2（主体+核心观点） | 平衡完整性和可读性 |

## 回复映射

| 用户回复 | 处理模式 | 说明 |
|----------|----------|------|
| A1 | full | 原文保存 |
| A2 / A | summary | 主体+核心观点 ⭐默认 |
| A3 | brief | 精简摘要 |
| B | individual | 进入逐条模式 |
| 1 | full | 逐条-原文保存 |
| 2 | summary | 逐条-主体+核心观点 |
| 3 | brief | 逐条-精简摘要 |
| Y / 空 | chat_keep | 保留聊天记录 |
| N | chat_delete | 删除聊天记录 |

## 免打扰逻辑

- 如果前一天无交互（无待读笔记、无聊天记录）
- 第二天早上的复盘报告静默跳过
