# 文件处理与信息提取策略

## 核心观点

在AI辅助的文件管理系统中，**保守策略**是首要原则：不确定时不删除、没记录时不删除、用户未明确时不删除。同时建立多层验证机制确保信息已安全提取。

## 关键讨论

### 1. 删除策略的演进

**用户原始需求：**
> "不是自动删除大于1M的文件，而是要看是否是文章截图，如果是，而且已经提取信息了，就删除"

**三层判断逻辑：**

```
┌─────────────────────────────────────────┐
│  条件1：文件大小 > 1MB                  │
│  （初步筛选可能的截图文件）               │
└─────────────────┬───────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  条件2：判断是否为文章截图               │
│  - 文件名含：screenshot/截图/image      │
│  - 或内容含：http/原文/作者/核心观点     │
└─────────────────┬───────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  条件3：确认信息已提取                   │
│  - 提取URL并计算hash                    │
│  - 检查笔记目录是否存在对应文件          │
└─────────────────┬───────────────────────┘
                  ↓
        三个条件都满足 → 安全删除
```

### 2. 文章截图识别机制

**文件名关键词检查：**
```python
screenshot_keywords = ['screenshot', '截图', 'image', 'img_', 'photo', 'pic_']
```

**内容特征检查：**
```python
article_markers = ['http', '原文', '作者', '来源', '文章', '阅读', 'key takeaway', '核心观点']
```

**信息提取验证：**
```python
# 从session内容中提取URL
url_match = re.search(r'https?://[^\s\n\r]+', session_content)
if url_match:
    url = url_match.group(0)
    url_hash = hashlib.md5(url.encode()).hexdigest()[:8]
    
    # 检查笔记是否存在
    for category in ['03-Articles', '03-Articles/Zhihu', ...]:
        for note_file in category_dir.glob(f"*_{url_hash}.md"):
            if note_file.exists():
                info_extracted = True
```

### 3. 保守策略原则

**用户明确指出的原则：**

| 情况 | 策略 |
|------|------|
| 不确定 | 不删除 |
| 没记录 | 不删除 |
| 用户没明确说删 | 不删除 |
| 无法确认信息已保存 | 跳过并记录日志 |

**日志记录示例：**
```
跳过非文章截图: xxx.jsonl (2.5MB)
跳过未确认提取: yyy.jsonl (无法确认信息已保存)
已删除文章截图: zzz.jsonl (1.2MB) -> 对应笔记已确认
```

## 有价值的思考

1. **删除是危险操作**
   - 数据一旦删除难以恢复
   - 宁可保守多保留，也不要误删
   - 建立多层确认机制

2. **信息提取的完整性验证**
   - 不只是检查文件是否存在
   - 要通过URL hash建立关联
   - 防止"假提取"（文件存在但内容为空）

3. **用户意图的准确理解**
   - 用户说"自动删除大于1M的文件" → 实际上是有条件的
   - 必须追问确认：是"只要是大的就删"还是"满足特定条件才删"
   - 主动列出边界情况、测试建议

## 实施细节

**代码实现关键点：**
```python
# 获取实际文件大小（而非内存占用）
file_size = file_path.stat().st_size

# 三条件判断
if file_size <= 1024 * 1024:
    continue  # 不满足条件1

if not is_article_screenshot:
    log(f"跳过非文章截图: {file_path.name}")
    skipped_count += 1
    continue  # 不满足条件2

if not info_extracted:
    log(f"跳过未确认提取: {file_path.name}")
    skipped_count += 1
    continue  # 不满足条件3

# 三个条件都满足 → 删除
try:
    file_path.unlink()
    log(f"已删除文章截图: {file_path.name}")
    deleted_count += 1
```
