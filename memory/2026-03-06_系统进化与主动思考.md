# 系统进化与主动思考机制

## 核心观点

用户与AI深入探讨了如何构建一个具备**自我进化能力**的AI系统，核心洞察是：**迭代次数决定智商**。通过建立规则沉淀机制，让AI从每次交互中学习并固化流程，实现持续改进。

## 关键讨论

### 1. 主动思考触发器的建立

**触发条件：**
- 用户反馈"被动"、"不够主动"
- 用户出现"你说呢"类反问
- 改代码/配置任务
- 复杂任务场景

**正确做法（规则6）：**
| 场景 | 行动 |
|------|------|
| 用户说"你说呢" | 直接给判断+完整方案，不问 |
| 改代码/配置 | 主动列出边界情况、测试建议、回滚方案 |
| 用户确认后 | 主动问"部署吗？"或"需要测试吗？" |
| 复杂任务 | 先给整体方案再动手，不边做边想 |

### 2. 自我进化机制设计

基于《和傅盛聊了3小时龙虾》的核心洞察，建立三层进化机制：

**① Cron规则 —— 龙虾的生物钟**
- 硬性规则：任何需要"记住做某事"的指令，必须明确说"写到Cron里"
- 原因：大语言模型有"以为自己做了"的幻觉

**② 迭代计数器 —— 龙虾的智商**
- 每次对话后问："这次有什么可以固化到流程中？"
- 记录每次重大改进：日期 + 改进内容 + 影响
- 每月回顾：迭代次数 vs 产出质量

**③ Agent.md自我进化 —— 犯错→反馈→写规则→不犯**
```
发现错误 → 分析根因 → 写入规则 → 用户确认 → 下次应用
```

## 有价值的思考

1. **员工思维 > 工具思维**
   - 把AI当员工培养，而非工具使用
   - 给完整权限（文件系统、GitHub、定时任务）
   - 犯错时反馈，而不是直接接手
   - 长期培养，越用越顺手

2. **保守策略原则**
   - 不确定 → 不删除
   - 没记录 → 不删除
   - 用户没明确说删 → 不删除

3. **三层记录体系**
   - **L1 原始记录**：自动捕获事件（ERRORS.md, LEARNINGS.md）
   - **L2 人工验证**：确认有效后推广到长期记忆
   - **L3 固化规则**：写入 AGENTS.md、SOUL.md 等

## 相关联想

- 与《龙虾养成日记》的关联：迭代次数决定智商
- 与 self-improving-agent 技能的对比：
  - self-improving-agent 是**自动捕获**
  - 本系统是**人工反馈+规则固化**
- 可以借鉴的"推广机制"：当某个规则被验证有效时，自动更新到更核心的位置
