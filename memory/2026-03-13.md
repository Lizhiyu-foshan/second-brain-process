## 🚨 重大事故：所有定时任务停止执行

### 问题发现

- **验证**: 健康检查报告停留在昨天 17:51，所有自动化任务未更新

### 根本原因
**OpenClaw 2026.2.13 版本更新导致 `isolated` + `agentTurn` 模式失效**

```
昨天：AI 生成文本 → 调用 exec → 真正执行脚本 → 更新文件
今天：AI 生成文本 → [无法调用 exec] → 只生成回复 → 文件不更新
```

`isolated` 会话中的 AI **无法调用任何工具**，只能生成文本。

### 影响范围
1. ❌ 文章剪藏链路健康检查（每10分钟）
2. ❌ 自动会话压缩（每10分钟）
3. ❌ 消息延迟监控（每10分钟）

5. ❌ 自我进化系统（检查已安装skill，标注来源）
6. ❌ 每日复盘报告（8:30）- **完全未执行**
   - 用户 9:01 发现未触发，手动提醒我补发
   - 9:18 的报告是**手动触发**，非自动任务

### 修复方案
**迁移到系统级 Linux Cron**（绕过 OpenClaw 定时任务系统）
