# 教训层 (Lessons)

记录踩过的坑、犯过的错，按严重程度分级。

---

## Critical（严重）

### 2026-03-17: 自动修复引入语法错误
**问题**: 看到审计报告说 CRITICAL 问题，直接运行自动修复脚本，破坏7个Python文件
**根因**: 没有人工验证问题真实性（实际是示例代码）
**后果**: 系统无法运行，最后从git恢复
**教训**: 任何修复都不能破坏原文件的运行逻辑，先思考后行动
**AGENTS.md**: 规则4 - 代码部署全流程

---

## High（高）

### 2026-03-08: 模型切换未检查上下文限制
**问题**: 从k2p5(262k)切换到GLM-5(128k)，上下文配置未更新
**后果**: `Range of input length should be [1, 202752]` 错误
**教训**: 切换模型时必须检查并调整上下文长度配置
**AGENTS.md**: 规则6 - 模型切换检查

---

## Medium（中）

### 2026-03-19: 静默任务发送了正常执行消息
**问题**: 动态上下文压缩等静默任务运行时，飞书收到"【定时任务-直接执行】..."消息
**根因**: 使用了 `systemEvent` + `sessionTarget: main`，systemEvent会把文本注入主会话历史，触发消息发送
**后果**: 静默任务不静默，用户被打扰
**教训**: 静默任务必须使用 `agentTurn` + `sessionTarget: isolated` + `delivery.mode: none`
**AGENTS.md**: 规则5 - 定时任务监控

### 2026-03-09: 定时任务未实际执行
**问题**: 美术馆展览推送任务显示"执行成功"但未发送消息
**根因**: `systemEvent`只注入文本，不会触发AI执行工具调用
**教训**: 需要AI执行工具调用的任务，要用明确的"执行命令"格式
**AGENTS.md**: 规则5 - 定时任务监控
