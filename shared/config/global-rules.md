# 全局共享规则 - 所有Agent必须遵守

## 用户核心偏好

### 工作模式
- **默认安全优先**：涉及删除/丢弃的操作，默认应保留而非删除
- **超时策略差异化**：
  - 聊天记录处理：5分钟超时（默认保留）
  - 待读笔记处理：15分钟超时（默认A2摘要模式）
- **GitHub推送**：四确认检查（当前目录、remote URL、当前分支、嵌套仓库）

### 模型使用规范
- **架构师Agent**：使用Kimi K2.5（深度思考、推理能力强）
- **开发者Agent**：使用GLM-5（复杂编码/架构设计）
- **测试员Agent**：使用Qwen 3.5 Plus（快速验证）

### 文件操作规范
- 使用 `trash` 替代 `rm`（可恢复优先）
- 重要操作前创建 `.backup/` 目录备份
- 配置修改前自动备份原文件

### 沟通风格
- 简短、有画面感，避免AI腔
- 给出明确判断，不绕弯子
- 一句话能讲清的事不拆成三段

## Agent协作规范

### 输入输出格式
- 所有Agent间传递的数据必须是JSON格式
- 文件编码统一使用UTF-8
- 时间戳格式：ISO 8601（2026-03-14T17:44:00+08:00）

### 错误处理
- 失败时必须输出明确的错误原因
- 保留中间状态文件便于调试
- 关键错误需要人工介入时，发送通知

### 资源清理
- 临时文件使用 `tmp_` 前缀
- 超过7天的临时文件自动清理
- 成功完成后保留报告文件，清理中间文件

## Pipeline协作流程

```
架构师输出 → shared/pipeline/plan_YYYYMMDD.json
开发者读取 ↑ 输出 → shared/pipeline/dev_report_YYYYMMDD.json
测试员读取 ↑ 输出 → shared/pipeline/test_report_YYYYMMDD.json
```

### 文件命名规范
- `plan_YYYYMMDD.json` - 架构师生成的改进计划
- `dev_report_YYYYMMDD.json` - 开发者生成的开发报告
- `test_report_YYYYMMDD.json` - 测试员生成的测试报告
- `*.log` - 执行日志
- `*.backup.json` - 自动备份

### 状态流转
1. `pending` - 等待处理
2. `processing` - 正在处理
3. `completed` - 完成
4. `failed` - 失败，需检查日志
5. `approved` - 测试通过，可部署

## 禁止事项

- ❌ 禁止直接修改其他Agent的工作目录
- ❌ 禁止跳过备份直接覆盖配置文件
- ❌ 禁止在生产环境运行未测试的代码
- ❌ 禁止忽略错误继续执行（必须显式处理）
