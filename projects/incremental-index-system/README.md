## 项目信息

- **项目名称**: Incremental Message Index System
- **创建时间**: 2026-03-21
- **优先级**: CRITICAL
- **风险等级**: HIGH

---

## 任务描述

开发增量消息索引系统，解决对话文件扫描的性能和完整性问题：

1. 维护消息级别的时间索引，记录处理到的最后消息时间戳
2. 每天只处理新增消息，避免全量扫描
3. 确保不遗漏任何消息（完整性）
4. 执行时间稳定在0.1秒内（稳定性）
5. 索引损坏时自动重建（可靠性）

---

## 全局约束

### 可靠性约束
- [ ] 索引损坏时自动重建，不丢失数据
- [ ] 支持异常恢复，断点续传
- [ ] 双重验证：索引时间戳 vs 实际消息时间戳
- [ ] 保留7天备份，防止误删除

### 稳定性约束
- [ ] 执行时间 < 1秒（无论文件数量多少）
- [ ] 内存使用 < 100MB
- [ ] 不阻塞定时任务
- [ ] 失败时优雅降级到原方案

### 完整性约束
- [ ] 不遗漏任何消息（100%覆盖）
- [ ] 正确处理跨天消息
- [ ] 正确处理乱序消息
- [ ] 正确处理重复消息

### 代码质量约束
- [ ] 单元测试覆盖率 > 80%
- [ ] 所有边界情况有测试用例
- [ ] 代码通过 AST 审计（无关键问题）
- [ ] 有详细注释和文档
