### 4. 错误处理策略

| 场景 | 处理策略 | 约束覆盖 |
|------|---------|---------|
| 索引文件不存在 | 触发首次全量重建 | R1 |
| 索引文件损坏 | 自动重建，从备份恢复 | R1, R2 |
| 校验和不匹配 | 重建索引，报警 | R3 |
| 扫描过程中断 | 下次从断点继续 | R2 |
| 文件读取失败 | 跳过该文件，记录日志 | S4 |
| 消息时间乱序 | 按时间戳排序处理 | I3 |
| 重复消息 | 基于时间戳+内容哈希去重 | I4 |

### 5. 性能优化策略

1. **快速检查**: 读取文件最后5行判断是否有新消息
2. **批量读取**: 使用生成器避免内存爆炸
3. **哈希去重**: MD5(content[:100]) 快速去重
4. **预分配**: 预估消息数量，预分配列表容量

## 决策记录

**设计决策**: 采用分层架构（IndexManager + Scanner）
- **原因**: 职责分离，便于测试和维护
- **替代方案**: 单体脚本（简单但不利于扩展）
- **风险**: 接口复杂度增加，但可控
